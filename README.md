# Axenta Geozones Service

Тестовое задание для позиции Junior Python Developer в Axenta.

Сервис позволяет:

- создавать геозоны
- получать список геозон
- проверять, попадает ли точка устройства в геозону
- сохранять результат проверки
- получать историю проверок с фильтрацией

По условиям задания геозоны хранятся в PostgreSQL/PostGIS, а проверка попадания точки в геозону выполняется средствами PostGIS, а не вручную в Python. 

## Стек

- Python 3.12
- Django
- Django REST Framework
- PostgreSQL + PostGIS
- Docker Compose
- drf-spectacular / Swagger
- ruff

## Формат геометрии и проверка попадания

### Формат геозоны

Для описания геозон используется WKT (Well-Known Text) формат.

Пример:

```
POLYGON((50.1200 53.1800, 50.1800 53.1800, 50.1800 53.2200, 50.1200 53.2200, 50.1200 53.1800))
```

Данный формат выбран, так как:
- он стандартный для PostGIS
- легко читается
- напрямую поддерживается в Django через `GEOSGeometry`

### Проверка попадания точки


Проверка выполняется на уровне базы данных (PostGIS) с использованием геометрического lookup:

```python
geometry__covers=point
```
